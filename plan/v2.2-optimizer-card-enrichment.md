# PRD — Optimizer suggested-lineup card enrichment

**Status:** Proposed (V2.2)
**Owner:** unassigned
**Related:** [PR #20](https://github.com/jhiggins-tech/2-OW-Live-Report/pull/20) (OverFast hero meta), [PR #24](https://github.com/jhiggins-tech/OW-Live-Report/pull/24) (hero portraits), [v2.md](./v2.md), [v2.2-overfast-player-endpoints.md](./v2.2-overfast-player-endpoints.md).

## Context

The Optimizer page's **Suggested lineup** section renders one card per
assigned player (`AssignmentCard` in `OptimizerPage.tsx`). Today each card
shows the player's name, role pill, rank label, KDA, win rate, and
optional lock indicator — all text. There's no visual identity per player,
no rank insignia, and no hint about *what heroes the player would actually
play in that role*. The optimizer has computed the role assignment but
hidden the per-hero data it used.

This PRD enriches each lineup card with:

1. **Player avatar** (head shot — already in InfluxDB `player_summary`).
2. **Rank insignia** for the assigned role (tier and division icons —
   **not** in InfluxDB; available from OverFast's `/players/{id}/summary`).
3. **Role icon** for the assigned role (Blizzard's canonical role
   insignia — also from OverFast's `/players/{id}/summary`), rendered
   alongside or replacing the existing text role pill.
4. **Top-5 heroes for the assigned role**, with portrait + win % + the
   player's pickrate of that hero within their role.

These are pure visual upgrades — the optimizer scoring itself doesn't
change.

## Goals

- Each suggested-lineup card renders the player's avatar at a noticeable
  size (e.g. 56×56, rounded).
- Each card shows the role icon (Blizzard's canonical tank/damage/support
  insignia) for the role the optimizer assigned, integrated with — or
  replacing — the existing text role pill.
- Each card shows the rank insignia for the assigned role (a `rank_icon`
  and ideally a `tier_icon` PNG from OverFast).
- Below the card body, the top-5 heroes in that role for that player are
  listed with: portrait, hero name, hero win %, player pickrate within
  the role.
- Graceful fallback whenever any of the four data sources isn't
  available — no broken images, no layout shift.

## Non-goals

- Changing the optimizer scoring function. Top-5 heroes is display only.
- Surfacing rank insignia anywhere outside the optimizer cards. The
  player profile header could reuse the same hook later, but that's a
  separate PR.
- Pickrate normalised against the team or the global meta. Player-local
  pickrate is the only metric in scope. (The team-meta overlay is its
  own PRD.)
- A bench/alternate view of unselected players. Just the five
  assignments.

## Data sources

| Field | Source | Already fetched? | Cost |
|---|---|---|---|
| `avatar`, `namecard` | InfluxDB `player_summary` (`fetchLatestPlayerProfile`) | Per player, on PlayerPage only | 1 statement per player |
| `rank_icon`, `tier_icon`, `role_icon` | OverFast `/players/{playerId}/summary` `.competitive.pc.{role}.{rank_icon, tier_icon, role_icon}` | No | 1 HTTPS request per visible player (shared across all three icons) |
| Per-hero stats (player) | InfluxDB `career_stats_*` already pulled in `fetchOptimizerData` | Yes — but rolled up to per-role and discarded | 0 |
| Hero portrait + name | OverFast `/heroes` via `useHeroMeta()` | Yes | 0 |

Confirmed by schema probe on 2026-05-11: `player_summary` fields are
{avatar, namecard, endorsement_level, endorsement_frame, title, username};
`competitive_rank` fields are {tier (float, division number), division
(string, tier name), season}. No `rank_icon` exists in InfluxDB. OverFast's
player summary returns canonical Blizzard CDN URLs for `rank_icon` and
`tier_icon` per role.

## Proposed approach

### Hook: `useTeamPlayerProfiles(players)`

New TanStack hook in `web-v2/src/hooks/useTeamPlayerProfiles.ts` that
fires one OverFast `/players/{playerId}/summary` per visible player in
parallel, with aggressive caching.

```ts
useQuery({
  queryKey: ['overfast', 'playerSummary', playerId],
  queryFn: () => fetchOverFastPlayerSummary(playerId),
  staleTime: ONE_HOUR,
  gcTime: 24 * ONE_HOUR,
  retry: 1,
});
```

Use `useQueries` (TanStack's batched variant) to issue all of them at
once and return `{ [playerId]: PlayerProfile | null }`. The OptimizerPage
component consumes the map and indexes by `a.player.playerId`.

The returned shape:

```ts
interface OverFastPlayerProfile {
  playerId: string;
  avatar: string | null;
  namecard: string | null;
  competitive: Partial<Record<Role, {
    division: string;        // 'silver', 'platinum', etc.
    tier: number;            // 1..5 division number
    rankIcon: string | null;
    tierIcon: string | null;
    roleIcon: string | null;
  }>>;
}
```

### Extend `fetchOptimizerData` to retain per-hero stats

Currently it aggregates per-(player, hero) data into per-role roll-ups
and discards the per-hero rows. Change to keep them so the UI can render
top-5 lists per role. Add to `PlayerOptimizerData`:

```ts
heroesByRole: Record<Role, Array<{
  key: string;
  prettyName: string;
  gamesPlayed: number;
  timePlayedSeconds: number;
  winRate: number | null;
  kda: number | null;
  pickrate: number; // share of role time, 0..100
}>>;
```

Sort each role's array by `gamesPlayed` desc (or `timePlayedSeconds` if
games_played is unreliable), top 5.

### Render

`AssignmentCard` becomes a two-row layout:

- **Header row.** Avatar (56×56, rounded) on the left; player name plus
  a role pill that now leads with the OverFast `role_icon` (18×18) and
  retains the existing tinted text behind it; locked badge in the
  middle; rank insignia stack on the right — `rank_icon` (28×28) with
  the `tier_icon` (20×20) overlaid or sat alongside.
- **Stats line.** Same as today (`rankLabel · KDA · win%`).
- **Hero list.** 5 rows, each: 20×20 portrait, hero name, win %,
  pickrate %. Compact — like a mini version of `HeroLeaderboard`.

CSS additions (~30 LOC in `theme/global.css`):

```css
.lineup-card { display: flex; flex-direction: column; gap: 10px; }
.lineup-card-header { display: flex; align-items: center; gap: 12px; }
.lineup-card .avatar { width: 56px; height: 56px; border-radius: 14px; }
.lineup-card .rank-insignia { display: flex; align-items: center; gap: 4px; }
.lineup-card .rank-insignia img { height: 28px; width: auto; }
.lineup-card .role-pill img { width: 18px; height: 18px; vertical-align: middle; margin-right: 6px; }
.lineup-card .hero-row { display: grid; grid-template-columns: 20px 1fr auto auto; gap: 8px; align-items: center; font-size: 0.85rem; }
```

### Fallbacks

- **OverFast outage / 4xx for a battle tag.** Card renders without
  rank insignia and without the OverFast-provided avatar — InfluxDB's
  `player_summary.avatar` is fetched as a backup. (Alternative: skip
  InfluxDB for avatar and just leave the slot blank if OverFast errors.
  Avoids running the extra query against InfluxDB. Default: skip the
  InfluxDB fetch unless avatar conspicuously matters in the no-OverFast
  case.)
- **Hero meta not loaded.** Hero list still renders with text hero
  names, no portraits.
- **Player has zero games in the assigned role.** "No recent games on
  this role" instead of the 5-hero list.

## Open questions

- **Pickrate denominator.** Is the player's pickrate of a hero relative
  to *their total playtime in that role*, or *their total playtime
  overall*? Role-local makes the numbers comparable across heroes
  shown in the same card. Default: role-local.
- **OverFast rate limit.** Tekrop publishes rate limits; 9 simultaneous
  player-summary requests on optimizer open is well under burst limits
  in practice, but worth confirming if the team grows. Each request is
  ~1.6 kB, so cost is small.
- **Avatar source: OverFast or InfluxDB.** Both have it. InfluxDB is
  cached by us for 5 min (sessionStorage); OverFast is cached for an
  hour. If we use OverFast, we cut the InfluxDB query for avatar; if
  we use InfluxDB, we avoid a network call per player but lose the
  rank insignia anyway. Default: OverFast for everything profile-level,
  since we're already calling it for insignia.
- **What if the optimizer's role assignment is `damage` but OverFast
  has no recent `damage` rank?** Fallback hierarchy: assigned role
  insignia → best role insignia → no insignia.
- **`tier_icon` over `rank_icon`.** OverFast returns both. `rank_icon`
  is the cross-tier badge (e.g. SilverTier); `tier_icon` shows the
  division number (1–5). Show both side-by-side, or composite into
  one element?
- **History.** The OverFast summary is a snapshot. The optimizer
  scoring already uses InfluxDB's historical per-role rank ordinal.
  We're only using OverFast for icons + avatar, so the snapshot
  mismatch (OverFast=live, InfluxDB=last-CI) is invisible to the user.
  Keep this in mind if anyone notices the rank label differing from
  what the insignia depicts. Acceptable for V2.2.

## Risks

- **Rate-limit on OverFast.** If we open the optimizer rapidly back-to-
  back the per-player summary queries may 429. Mitigation: 1-hour
  staleTime in the hook (already in the design); retry once on failure;
  silent graceful degrade.
- **Card height blow-up.** Adding the avatar + insignia + 5-hero list
  takes the card from ~4 lines to ~12+ lines. Lineup grid is currently
  `cols-3`; on a wide screen this still fits. On narrow screens the
  grid collapses to 1 column and the cards become tall but readable.
- **Stale OverFast data.** If a player ranks up between the last
  CI run and the OverFast fetch, the insignia and the rank label
  disagree. Acceptable. (See open question above.)
- **Per-hero data dropping out.** If a player has only 1–2 games on
  a role, the top-5 is shorter than 5. Render however many are
  present rather than blank rows.

## Acceptance criteria

- Each suggested-lineup card has an avatar that's clearly visible
  (≥56 px), the player's name, a role pill leading with the OverFast
  `role_icon`, a rank insignia (`rank_icon` + `tier_icon`) for the
  assigned role, the existing stats summary, and a 5-hero list with
  portraits, name, win %, and pickrate.
- OverFast outage: cards still render with avatar (from InfluxDB
  fallback) and a text-only role pill (no role / rank insignia), no
  broken images.
- Hero meta cache miss: hero list renders text-only with same numbers.
- A player with zero games in the assigned role shows a "no recent
  games" placeholder instead of an empty list.
- No new InfluxDB statements (the per-hero data is already fetched
  by `fetchOptimizerData`). One OverFast HTTPS request per visible
  player, with 1-hour staleTime.
- Tested manually against the live site after deploy.
